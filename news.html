Intel CPU Bugs: Meltdown and Spectre Okay so lets get the FUD out of the way. 4 days before the news about the bug could have gotten out (because thats when it would be posted to the Linux kernel developer mailing list), Intel's CEO sold every share in his company that he was legally allowed to share (source: http://www.businessinsider.com/intel-ceo-krzanich-sold-shares-after-company-was-informed-of-chip-flaw-2018-1). Finally, two lines of code were added to the linux kernel basically saying "if the CPU vendor is not AMD, then it is insecure". Intels stock went down substantially when the news broke, and AMD's stock went up substantially. Also, pretty much everyone hates Intel right now because they made a whole bunch of devs work a lot of overtime over new years to fix all this. Okay, with that out of the way the rest of this recap will be about the technical fuckups, what you need to do to fix it, and what impact that will have. 1. How did this happen? Intel CPU's use speculative execution to speed up execution. This is where the CPU guesses what the running program will do next, and does that very quickly. It then checks if that was the right action. If it is right (which it is a good enough percent of the time to make this actually speed up the CPU), then it continues as normal, otherwise it undoes the action and does the correct one. So wheres the problem? Theres certain things the CPU wont or cant undo. So if the guessing in the CPU is made to access memory it shouldnt be allowed to, and save that value into the CPU cache (which the CPU wont undo changes to because that would break things), then even after the program is caught reading memory it isnt supposed to, and then it is "undone", it still has the information saved in the CPU cache, which it may be able to read. The other exploit is similar but not quite the same. If you want to read the technical breakdowns, heres the links: https://meltdownattack.com/meltdown.pdf https://spectreattack.com/spectre.pdf And a somewhat less technical breakdown: https://www.theverge.com/2018/1/6/16854668/meltdown-spectre-hack-explained-bank-heist-analogy 2. I recommend not using antivirus anyway...). You should also patch if you dont have an Intel CPU, because AMD and ARM CPUs were also (to a much, much lesser extent) affected by some speculative execution bugs. You should also keep an eye out for firmware patches in the coming weeks, as Intel has promised to provide firmware patches to hopefully also disarm Specture and Meltdown. https://www.bleepingcomputer.com/news/hardware/intel-promises-firmware-updates-for-most-modern-cpus-by-the-end-of-next-week/ 3.
